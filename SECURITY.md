# Security Policy

We take the security of the VerifNow Java SDK and the VerifNow platform seriously.

If you believe you have found a security vulnerability, **do not open a GitHub issue**.  
Instead, please follow the steps below.

---

## Reporting a Vulnerability

If you discover a potential security issue, please contact us at:

We request responsible disclosure and allow our team time to review and address the issue before it is made public.

## Vulnerability Handling Process

1. **Report received** via `support@verifnow.io`
2. **Initial review** and reproduction attempt
3. **Classification** (severity and impact)
4. **Fix development** and internal review
5. **Coordinated disclosure** with the reporter
6. **Release** of a patched version + public advisory
